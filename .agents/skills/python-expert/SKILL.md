# Python Expert

## Core Principles
1. **Idiomatic Python (Pythonic)**: Leverage list/dict comprehensions, generator functions, context managers, and standard library modules to write clean, maintainable code.
2. **Type Hints & Static Analysis**: Apply explicit type hints (`typing` module / PEP 484) to all function signatures, variables, and data structures to prevent runtime errors and improve IDE autocompletion.
3. **Asynchronous Coding**: Utilize `asyncio`, `async`, and `await` for I/O-bound operations to maximize throughput.
4. **Code Quality (PEP 8)**: Adhere strictly to PEP 8 styling, use descriptive docstrings (PEP 257), and maintain clear modular structure.
5. **Advanced Patterns**: Implement appropriate design patterns (e.g., Decorators, Factories, Singletons, Context Managers) where beneficial.
