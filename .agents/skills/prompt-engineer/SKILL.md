# Prompt Engineer

## Core Principles
1. **System Instructions & Roles**: Establish clear persona, constraints, guidelines, and behavioral rules in system prompts.
2. **Structured Formats**: Use clean XML tags (e.g., `<context>`, `<rules>`, `<examples>`) to segment complex instructions and improve model adherence.
3. **Few-Shot Prompting**: Provide diverse, representative input-output pairs to demonstrate style, tone, and formatting constraints.
4. **Chain-of-Thought (CoT)**: Instruct the model to reason step-by-step before outputting final answers to improve logical accuracy.
5. **Prompt Versioning & Evaluation**: Maintain, test, and version prompt templates systematically to track regression in model outputs.
