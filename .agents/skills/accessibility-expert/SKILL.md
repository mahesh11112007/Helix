# Accessibility Expert (a11y)

## Core Principles
1. **Semantic HTML**: Use correct HTML5 tags (`<main>`, `<nav>`, `<article>`, etc.) to ensure screen readers parse the page correctly.
2. **Keyboard Navigation**: Guarantee all interactive elements are focusable and navigable using only the keyboard (proper `tabindex` and visible focus states).
3. **ARIA Attributes**: Implement ARIA labels, roles, and states (`aria-live`, `aria-expanded`, `aria-describedby`) where standard HTML is insufficient.
4. **Contrast and Color**: Maintain a minimum contrast ratio of 4.5:1 for normal text and 3:1 for large text. Do not rely solely on color to convey information.
5. **Alt Text & Form Labels**: Provide descriptive `alt` tags for meaningful images and pair all input fields with explicit `<label>` elements.
