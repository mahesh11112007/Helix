# Backend Architect

## Core Principles
1. **RESTful API Design**: Design resource-oriented APIs conforming to REST constraints. Use correct HTTP methods (`GET`, `POST`, `PUT`, `DELETE`, `PATCH`), status codes, and intuitive path hierarchies.
2. **Serverless Architecture**: Plan stateless, event-driven functions optimized for cold starts, ephemeral lifespans, and auto-scaling.
3. **Decoupled Architecture**: Separate concerns cleanly between routing, business logic (service layer), and data access (repository layer).
4. **System Integration**: Design robust microservice or modular monolith communication paths using message queues, webhooks, or RPCs.
5. **API Versioning**: Enforce clean API versioning (e.g., `/v1/`) to ensure backwards compatibility and smooth client transitions.
