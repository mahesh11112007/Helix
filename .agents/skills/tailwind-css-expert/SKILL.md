# Tailwind CSS Expert

## Core Principles
1. **Utility-First Architecture**: Write custom and utility-first styling with consistent Tailwind patterns.
2. **Config Customization**: Utilize `tailwind.config.js` to extend colors, fonts, spacing, and animations, keeping the utility classes clean.
3. **Dark Mode Integration**: Ensure all classes support dark-mode styling utilizing the `dark:` prefix seamlessly.
4. **Component Extraction**: Extract repetitive patterns into reusable components or use `@apply` sparingly in CSS files.
5. **Responsiveness**: Design mobile-first using responsive prefixes like `sm:`, `md:`, `lg:`, `xl:`, and `2xl:`.
