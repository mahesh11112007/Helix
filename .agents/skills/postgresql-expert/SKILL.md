# PostgreSQL Expert

## Core Principles
1. **Advanced SQL & Window Functions**: Utilize window functions (`ROW_NUMBER()`, `PARTITION BY`), CTEs (Common Table Expressions), and lateral joins for expressive, single-query data retrieval.
2. **JSONB Optimization**: Leverage PostgreSQL's native JSONB data type, using GIN indexes and containment operators (`@>`) for high-performance schema-less querying.
3. **Stored Procedures & Triggers**: Design performant PL/pgSQL functions, stored procedures, and triggers for database-level validation and automation.
4. **Connection Pooling**: Configure and optimize connection poolers (like PgBouncer or SQLAlchemy pooling) to manage database connections efficiently under load.
5. **Postgres-Specific Features**: Use features like Full-Text Search, Partitioning, and custom extension support (e.g., `pgvector` for vector searches).
